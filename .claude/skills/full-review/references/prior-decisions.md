# 이전 전수 검토에서 확정된 결정사항

> 이 문서의 내용은 이미 사용자와 합의된 사항이다. 에이전트들은 이 결정을 다시 논쟁하지 말고, 이를 전제로 새로운 발견에 집중하라.

## 확정된 제품 원칙

1. **카톡 대체 서비스**: "카톡으로 하면 된다"는 논거 금지. 이 서비스가 카톡을 대체해야 한다. 투표, 공지, 일정, 양도, 소통 모두 앱 안에서 완결.
2. **선택적 기록 깊이**: 기록 능력이 있는 팀은 개별 투구(PitchEvent)까지 기록 가능. 없는 팀은 기본 기록만. 강제하지 않으면 됨.

## 확정된 삭제 대상

| 대상 | 결정 | 사유 |
|------|------|------|
| SpecialGameRecord (노히트/퍼펙트 감지) | 삭제 | 5~7이닝 사회인 야구에서 발생 확률 ≈ 0 |
| Election + Emergency (선거/비상대책위) | 삭제 | transferOwnership() 메서드로 대체 (10-20줄) |
| Certificate (기록 증명서 QR) | 삭제 | 사회인 야구에서 불필요 |
| StandingsSimulation (몬테카를로) | 삭제 | 8팀 20경기에 1000회 시뮬레이션 과잉 |
| PitchingDecisionCalculator | 삭제 | @Deprecated 명시, 394줄 죽은 코드 |
| BookingTransfer (구장 예약 양도 거래) | 삭제 후 단순화 | 상태머신+가격 과잉. 단순 양도 기능으로 재설계 (앱 내) |
| PlayerTeamHistory.transfer() + TRANSFERRED | 삭제 | 사회인 야구에 이적 없음. ACTIVE/INACTIVE만 유지 |
| DOUBLE_ELIMINATION | 삭제 | 사회인 야구 대회에서 미사용 |

## 확정된 유지/리디자인 대상

| 대상 | 결정 | 사유 |
|------|------|------|
| PitchEvent (개별 투구) | 유지 (선택적 기능) | 타자별 구종 분석 가치. 안 쓰면 0행이라 DB 부하 없음 |
| CorrectionRequest (기록 정정) | 유지 | 비전문 기록원 실수 대응. Appeal 기능 흡수 통합 |
| GameRules | 유지 | 대회마다 규칙이 다름. 유연성 필수 |
| CareerStats | 유지 | O(1) 캐시 역할. finalize/unfinalize만 삭제 |
| Discipline → PlayerBan | 극단 축소 | BAN만 유지, 5필드 최소 엔티티 |
| ActivityScore | 조건부 삭제 | 수동 PUT 삭제, 경기참여율 자동 집계만 유지 |

## 확정된 개념 변경

| 기존 개념 | 올바른 개념 |
|----------|-----------|
| 용병 = 팀에 임시 선수 빌려오기 | 용병 = 팀 상관 없이 야구하고 싶은 개인들이 모여 이벤트 게임/픽업 게임 |
| 이적 = PlayerTeamStatus.TRANSFERRED | 이적 개념 없음. 탈퇴 + 새 팀 가입 |
| GameParticipation (경기 전용) + AttendancePoll (범용) 두 벌 | 투표 시스템 한 벌 통합 + 이벤트 카테고리 (경기/연습/회식/모임/기타) |
| 출석 투표 수동 생성 | 경기 1주일 전 자동 생성 + 하루 전 자동 마감 |
| 구장 양도는 카톡으로 | 구장 양도도 앱 내 처리 (단순화된 형태) |
